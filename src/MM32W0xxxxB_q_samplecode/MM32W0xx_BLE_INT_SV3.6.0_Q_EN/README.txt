1. Procedure description:
     The BLE library runs as a system interrupt service program, which is suitable for application scenarios where a user's 
function requires a long CPU time but can be interrupted. Need to use two interrupt service routines,
     One is the GPIO interrupt corresponding to the IRQ interrupt pin, and the other is the interrupt corresponding to the SysTick. 
The IRQ corresponding interrupt service routine is used to run the Bluetooth protocol Interrupt priority (for other user interrupts).

////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////// 
//////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
Hardware description:
    In conjunction with the Bluetooth protocol stack, system applications implement interfaces such as SPI communication and timers.
 The main functions include:1.unsigned char SPI_WriteBuf (unsigned char reg, unsigned char const * pBuf, unsigned char len);

2.unsigned char SPI_ReadBuf (unsigned char reg, unsigned char * pBuf, unsigned char len);
  The above two functions are implemented in the spi.c file and are related to the communication of BLE, please do not modify it. 
To ensure BLE operation, the spi clock should be greater than 6MHz and less than 10MHz.
  
3. Char IsIrqEnabled (void); Determine whether IRQ signal generates interrupt (low level is interrupt effective).

4.void IRQ_RF (void);
The above two functions are implemented in the irq_rf.c file and are related to the communication of BLE, please do not modify it.

5. unsigned int GetSysTickCount (void); Get the accumulated value of the millisecond timer, used for timing and other functions.

6, void SysTick_Handler (void)
The above two functions are implemented in the bsp.c file and are related to BLE communication. Please do not modify them.
 The accuracy of the system tick timer can be changed, such as from 1ms to 10ms, without affecting the normal operation of BLE.

7. Example of low power consumption function implementation:
void IrqMcuGotoSleepAndWakeup (void)
{
   if (ble_run_interrupt_McuCanSleep ())
   {
     // to do MCU sleep / stop / standby
   }
} This function is called cyclically in the user program of the main () function.
